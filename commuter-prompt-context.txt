Tech Stack & Rules:

Framework: Next.js 14+ (App Router), TypeScript, Tailwind CSS.
Theme: Strictly Dark Mode. Backgrounds: #050F1A (deepest), #071A2E (cards/surfaces), #1A5FB4 (primary blue), #62A0EA (light blue accents), #FF6D3A (orange action buttons). Text: text-white, text-white/50, text-white/30.

Component Architecture: Reusable components belong in components/commuter/[feature]/. Page-specific logic stays in app/(commuter)/[feature]/.
Layout Structure: The app uses a root layout at app/(commuter)/layout.tsx containing a desktop sidebar and mobile bottom navigation. Modules (Dashboard, Lost & Found, Rewards, Feedback) are sibling folders inside app/(commuter)/.
Current Modules Built:

Dashboard (/dashboard): Contains an interactive Leaflet map (dark CartoDB tiles) showing the Calumpit-Meycauayan route (ROUTE_COORDS). Has a sliding bottom sheet (mobile) / right-side panel (desktop) for wallet balance, Quick Actions (Show QR, Fare Calc, Share Ride, SOS), and a "Hail Me" button.
Lost & Found (/lost-and-found): Displays items posted by Admins. Features pagination (15 items/page), dynamic category filtering, a Watchlist (save for later), and a Claim system. Claiming requires a "Proof of Ownership" text input. Commuters are limited to 3 active pending claims. Uses leaflet-geometryutil for spatial queries (if needed).
Rewards (/rewards): Loyalty system. Shows a circular progress ring for rides (e.g., 7/10 rides). Base fare logic is ₱18 for 4km, ₱2 succeeding. 20% discount for Students/Seniors/PWD. Features a "Free Ride" voucher system with a dynamic QR code modal.
Fare Calculator (fare-calc-modal.tsx): Replaced dropdowns with an interactive map. Commuters drop a pin on the route; the app snaps the pin to the nearest route coordinate using leaflet-geometryutil. Calculates distance and fare. Crucial Flow: Payment requires scanning a physical Unit QR code. The UI simulates a camera scanner. The scanned unitId + routeIndex is sent to the backend to validate the fare and assign the transaction to the correct driver.
Feedback (/feedback): Opens the camera scanner immediately to scan a Unit QR. Loads the Driver/Conductor assigned to that unit. Commuters rate 1-5 stars. 1-3 stars shows negative tags (Overspeeding, Rough Driving, Distracted Driving, Overloading). 4-5 stars shows positive tags. Has a 3-second circular countdown auto-close success modal.
